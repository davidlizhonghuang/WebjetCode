Scope of the project

This is a simple front end web application. Web site should be able to list the all movies as a list. When user clicks each image in the movie list, the selected movie details should be displayed. The web page should be responsive for multiple devices.

Development

Back end service

Client provides two APIs' links we can use to retrieve and fetch all movie lists and each movie detail.

Front end web page layout

page should have a header to display logo, a content section to display main content, and a footer.

Movie list should be listed as a side menu located in the left side of the page via API calling one of the APIs. When user clicks each movie in the list, another API is used to retrieve the detail of the movie and display all details in the right side of the page. The layout should be responsive in multiple devices.

Click button to retrieve the lowest price of the movie

AngularJS

Angular JS 1.5 is the excellent front end framework we can use for SPA development. header, content, and footer in the page can be developed as components in landing page as example here

Due to the CORS that needs the data service layer to enable its CORS in server side to allow my web site access, we can not use angular js http call and even the jquery ajax call to return the data from APIs . However, we still can use POSTMAN in chrome to retrieve all data we need in both APIs and build fake routine here to mock the data interaction.

Project constraints

Oauth2 asp.net web APIs 2 for this testing does not enable their CORS to support this web site access. Therefore, I downloaded the data from POSTMAN and embedded them into the code as a dummy data for the SPA via the following links in POSTMAN

Summary

Angular Js 1.5 allows us to develop a web site based on the component design pattern. Inter-component communication is important for this success. Jasmine unit test is necessary for angular component development.
